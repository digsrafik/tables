× Minimal table width is 560px
  • Up to this point paging and items on page are next to each other.
  • If header labels are too long, allow wrapping. If wrapped text has more than 3 lines, truncate it and add elipsis.
  • If table content is too long (too many columns) allow scrolling and add property for the table to have first column fixed and scroll only the next columns.
Need narrower table?
  • Use listing (up to 2 columns).
  • In case you need more columns, use table and move place it to wider space.
× Sorting items
  • Arrow indicates which and how the column is sorted.
  • On cursor hover column header shows a tooltip. The cursor displayed is pointer.
  • Default sorting is something that depends on usage of the grid component. (Not always first column.)
  • To sort different column click its title. How it is sorted depends on data in the column: COLUMN WITH TEXT: 1st click = A-Z. 2nd click = Z-A. COLUMN WITH NUMBER: 1st click = highest-lowest. 2nd click = lowest-highest.
Items per page
  • Appears once there are more than 10 items in the grid.
  • Default number of items in the grid is something that depends on usage of the grid component.
  • Options are 5, 10, 25, 50 and 100 and (all).
Paging
  • Appears only when needed. Whether it's needed depends on "Items per page" option selected.
  • The maximum number of pages shown in the paging is 5. In case there are more pages than 5, elipsis and page selector (dropdown) is displayed.
Expanding rows
  • Only one level can be expanded.
  • The toggle is only the icon, not the row.
Alignment
  • Numbers to the right.
  • Text to the left.
